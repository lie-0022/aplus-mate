## 🎬 추천 둘러보기 순서

### ① 교수 현황 대시보드 — "내 수업이 어떻게 돌아가나" (먼저 보기)
운영자(레이) 계정 → **교수 페이지 → 수업 "소프트웨어 캡스톤 디자인" 선택 → 현황 탭**
- 수강생 6명 / 프로필 완성 6명
- **진행 중 팀 1개**, 팀 합류 3명 / **팀 미배정 3명**(콕 집어 보임 → 독려 대상)
- **설문 응답률** 막대(4/6명, 67%)
- 마감 임박 산출물 안내
> 👉 *교수님이 로그인하면 가장 먼저 보는 화면. "한눈에 파악"이 핵심입니다.*

### ② 수강생 명단 — 실명·강점 보기
**교수 페이지 → 수강생 탭**: 6명의 이름·학과·학년·할 줄 아는 것(스킬 태그)

### ③ 팀 현황 → 팀 속으로
**교수 페이지 → 팀 탭**: 3인 팀(김민준·이서연·박지호) 구성·진행상태
→ 학생 화면의 **팀 상세**에서 보이는 것: **일정**(담당자·D-day·진척도 막대), **팀 메모**(역할 분담 기록), **AI 보고서 초안**, **산출물 제출**

### ④ 설문 결과 분석
**교수 페이지 → 설문 탭 → "중간 강의 만족도 조사" 결과 보기**
- 5점 척도 **평균·분포 그래프**, 객관식 집계, **주관식 답변** 익명 나열
- **CSV 내려받기** 버튼

### ⑤ 산출물 제출 현황 — "걷는 수고" 없이
**교수 페이지 → 제출 탭**: '1차 기획안' 마일스톤에 **어느 팀이 냈는지 표로**, 제출 링크 열어보고 **"확인" 체크**

### ⑥ 공지
**교수 페이지 → 공지 탭**: '중간고사 안내' (작성하면 수강생 전원에게 알림)

### ⑦ 학생 입장 — 매칭·게시판·알림
- **게시판**(수업 화면): '알고리즘 스터디 모집', '지난 학기 자료' 글 + 익명 댓글·신고
- **매칭**: 데모 학생끼리 이미 팀이 맺어졌고, **대기 중인 스터디 요청**도 있어 매칭 흐름의 결과를 볼 수 있음
- **알림센터**(헤더 종 아이콘): 매칭 수락·새 공지 알림

## 📦 데모에 깔린 것 (기능별 정리)

| 기능 | 깔아둔 데이터 |
|---|---|
| **예시 수업** | 소프트웨어 캡스톤 디자인 (담당: 레이) |
| **학생 6명** | 김민준(컴공3·React/Node/발표), 이서연(디자인3·Figma/UIUX/기획), 박지호(경영2·기획/PPT/발표), 최유진(컴공4·Python/데이터분석/ML), 정현우(전자3·C++/임베디드), 강수아(경영2·마케팅/기획/영어) |
| **팀플 팀(3인)** | 김민준·이서연·박지호 — 일정 2개(담당자 지정), 팀 메모(역할 분담), 1차 기획안 제출 |
| **미배정 학생** | 최유진·정현우·강수아 (교수 대시보드에 독려 대상으로 표시) |
| **설문** | 중간 강의 만족도(척도2·객관식1·주관식1), 4명 응답 |
| **공지** | 중간고사 안내 |
| **게시판** | 스터디 모집 글, 지난 학기 자료 글 |
| **대기 매칭** | 최유진 → 정현우 스터디 요청(미수락 — 받은 요청 체험용) |

## 👤 계정 안내
- **교수 기능 보기**: 레이(운영자) 계정이 이 수업 담당 교수 → 레이로 교수 페이지 진입. 또는 교수님 계정을 운영자 페이지에서 "교수"로 지정.
- **학생 입장 직접 체험(커넥트→수락)**: 현재 데모 수업이 "데모대학교" 소속이라, 직접 매칭을 눌러보시려면 같은 대학으로 맞춰 재생성이 필요합니다(요청 시 즉시 처리).
