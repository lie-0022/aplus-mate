# A+ Mate 시간표 도입 설계 (다음 학기 실데이터 적용 대비)

> 2026-06-15 분석. **구현은 다음 학기 시간표가 나오면 착수**한다. 지금 참고한 PDF(백석대 2025-2 교양)는 옛날 데이터·형식 참고용. 멀티에이전트 분석 + pdfplumber PoC로 실현성·설계를 확정해 둔 청사진.

## 확정된 전제(사용자 결정)
- **데이터 소스**: 미정 — 학교 교무처 엑셀/CSV 입수 가능 여부 확인 필요. 받으면 파싱 대부분 회피. 안 되면 PDF 파싱(아래). → 파서는 **PDF/엑셀 양쪽 대응**으로 설계.
- **매칭 단위**: **타입별 차등** — 팀플(project)=분반 고정, 스터디·멘토링=같은 과목 전체. → **`courseGroupId`(과목 그룹키) 필요**.
- **범위**: **교양 + 전공** — 파서는 교양 전용이 아니라 범용. 전공 시간표는 형식이 다를 수 있어 입수 시 재검증.
- **시점**: 실데이터는 다음 학기 시간표 공개 후 적재.

## PoC 결과 — 파싱 실현성 입증 (`scripts/parse_timetable_poc.py`)
- PDF는 **텍스트 레이어 있는 디지털 문서**(스캔본 아님 → OCR 불필요).
- **괘선이 있어 `pdfplumber.extract_tables()`가 15개 컬럼을 자동 분리.** pypdf `extract_text()`의 "붙은 텍스트" 문제가 해소됨 — 거대 정규식 분해 불필요.
- 데이터 행 예: `['교필','기독교인성','교양','사랑의실천','3','0335501','기독교세계관','1','1','40','김호영','목7','진리406','사회복지학부','섬김(100)']` — 시수·학점·인원·교수·요일교시·강의실 전부 별도 컬럼.
- 규모: 교양만 **1,227강의**(교필 883 / 교선 352, 분반까지 별개). → **수동 입력 불가, 자동 파싱 + 검수 필수.**
- 후처리 보정 필요: 권장학년/과목코드 컬럼 경계 미세 어긋남(`'3','0335501'`→과목코드 `30335501`), 2017/2018 이수영역 서브컬럼 병합, 교수 글자사이 공백.

## 데이터 모델 변경안

### courses (수정)
- **유니크 교체**: `(name, professor, university)` → **`(courseCode, university, semester)`** — 현행 제약이 다분반 적재의 1차 장애물(같은 교수·같은 과목명 2분반 = ER_DUP_ENTRY). 과목코드(7~8자리)가 분반 자연키.
- **`semester` 추가**(현재 userCourses에만 있음) — 시간표는 학기별이라 courses에 고정.
- **컬럼 추가**: `campus`, `courseType`(enum 교필/교선), `areaPre2018`, `areaPost2018`, `recommendedYear`(varchar — '전체' 때문), `capacity`(int), `hours`(int 시수), `note`(text 특이사항), `competencies`(json `{name,ratio}[]`), `rawSchedule`(text 원문), `sourceKey`(과목코드+학기, 멱등 upsert 키).
- **`courseGroupId`**(과목명+학교 그룹키) — 타입별 매칭 차등용(스터디·멘토링이 분반 넘어 매칭).
- `courses.create`를 **adminProcedure로 제한**(현재 학생도 생성 가능 — 시간표 도입 후엔 운영자 일괄 적재).

### course_schedules (신규 — 1수업:N교시)
`{ id, courseId(FK), dayOfWeek mysqlEnum['월','화','수','목','금','토','사이버'], period int nullable, room varchar }`
- 연강 `목2,3` → (목,2)·(목,3) 2 row. 사이버혼합 `사/ 화3` → (사이버,null)·(화,3) 2 row.
- 공강 비트마스크·시간충돌·격자 렌더의 데이터 기반.

### 매칭 호환성
분반=courses row로 분리하면 매칭 외래키(courseId)는 **무수정**. team_matches/teams/posts/surveys 전부 그대로. 매칭이 자동으로 분반 단위가 되어 정밀해짐. 타입별 차등은 createMatchRequest/acceptMatch에서 project=courseId, study/mentoring=courseGroupId 기준으로 분기.

## 파싱·시딩 파이프라인 (PDF 경로)
1. (우선) 교무처 엑셀/CSV 입수 시도 → 받으면 컬럼 매핑만.
2. `pdfplumber.extract_tables(vertical_strategy='lines')` 좌표 추출 → 15컬럼 행.
3. 셀 단위 후처리: 과목코드 경계 보정, 시수·학점·인원 분해(이미 분리됨), 요일교시 파싱(연강 리스트·사이버 분리), 교수 공백 제거(복수교수 콤마 보존), 이수영역 서브컬럼 병합.
4. 자동 불변식 검증(코드 7~8자리·시수학점 1~6·인원 1~300·역량비율 합100·이수구분 enum·필드수). 실패행만 LLM 폴백(추측 금지·불명확 null).
5. JSON + `_raw` + 검증 리포트 산출 → **구글시트로 사람 검수**(1코호트엔 Admin 적재 UI 안 만듦).
6. `sourceKey` 기준 **멱등 upsert** 일회성 시딩 스크립트로 적재.
7. 표본 50~100행 원본 수기 대조 95%+ 확인.

## 시간표로 가능해지는 신기능 (놓쳤던 것, 효용순)
| 기능 | 가치 | 난이도 |
|---|---|---|
| **공강 매칭** | 팀원 공통 빈 시간(주간 점유 비트마스크 AND-NOT)으로 후보 정렬 — 팀플 최대 페인 해결, 차별점 | 중 |
| **시간 충돌 경고** | 수강등록 시 내 시간표와 겹치면 경고(막지 않음) | 쉬움 |
| **내 시간표 격자 뷰** | 요일×교시 격자 + 공유 — 에브리타임식 후킹·바이럴 | 중 |
| **검색 필터** | semester(기본)·campus·courseType·credits·dayOfWeek — 분반 폭증 대응 필수 | 중 |
| **매칭 범위 확장** | courseGroupId로 같은 과목 다른 분반 통합(study/mentoring) | 어려움 |
| 졸업요건/이수영역 집계 | 역량기반 교양 학교 차별점·제휴 포인트 | 어려움 |
