## 기존 → 수정본 (디자인 결정)

### 공통
- **수업 헤더**: `border+shadow-none` → `bg-card shadow-card`. `팀플` 그라데이션 배지 → **중립 알약 태그**. `교수님 인증`을 보라칩 → **인증 체크칩**(중립 그린 계열, 브랜드 보라 제거).

### 정보 탭
- **공지**: amber 하드코딩(`bg-amber-50/border-amber-200`)이 다크에서 뜨는 문제 → **notice 토큰**(라이트 소프트 피치 / 다크 반투명)으로 다크 안전화 + 스피커 아이콘.
- **설문**: `border-primary/30` 아웃라인 → 은은한 라벤더 `secondary` 카드 + 솔리드 참여 버튼.
- **카테고리 필터**: 아웃라인+그라데이션 버튼 → **테두리 없는 알약 칩**(활성=솔리드, 비활성=중립).
- **글쓰기**: outline → 소프트 `secondary`.
- **게시글 카드**: `border shadow-none`+아웃라인 카테고리 → `bg-card shadow-card`+중립 알약 카테고리.
- **빈 상태**: 점선 카드 → 카드화 + 2줄 카피 + 글쓰기 버튼.

### 팀원 찾기 탭
- **커넥트 종류**(팀플/스터디/멘토·멘티): 아웃라인+그라데이션 → **세그먼트 컨트롤**(탭식, 테두리 없음).
- **모집공고**: 밋밋한 border 카드 → 라벤더 `secondary` 패널 + 정돈된 항목 카드.
- **학생 카드**: `공통 스킬` 보라칩 → **긍정 그린칩**, 매칭 스킬 풀보라 → **소프트 라벤더**, 나머지 중립 알약. 커넥트 버튼 그라데이션 → 솔리드.
- **미등록 CTA**: 점선 → 카드화.

### PC (메인+우측레일)
- 기존은 모바일 콘텐츠를 좁게 가운데 정렬("너무 모바일틱") → 수정본은 **메인(1.6fr) + 우측 레일(1fr)**:
  - 정보 탭: 메인=카테고리 칩+게시글 **2열 그리드**, 레일=수업 소식(공지·설문)+글쓰기.
  - 팀원 탭: 메인=학생 카드 **1열**(내용 많아 넓게), 레일=커넥트 종류+안내+초대 링크 복사.

## QA — 목업에서 잡은 문제 & 수정 (2026-07-06, 크롬 실측)

사용자 지적: "수정본에서 깨지는 게 많다 — 글자/아이콘 크기, 사이드바 등등."

| # | 문제 | 원인 | 수정 |
|---|---|---|---|
| 1 | **PC 사이드바가 위에 얹히고 본문이 아래로 밀림** (라이트·다크·양 탭 전부) | 목업 CSS **`.pbody` 클래스 이름 충돌** — PC 프레임 본문(`display:flex`)을 게시글 텍스트(`display:-webkit-box;line-clamp`)가 뒤에서 덮어써 `flow-root`가 됨 | 게시글 텍스트 클래스 `.pbody`→`.pexc` 개명. 프레임이 다시 flex → 사이드바 전체높이·본문 정위치 |
| 2 | **"글쓰/기" 줄바꿈** (모바일 정보 필터행) | 알약 칩이 폭을 잠식, 버튼 내부 텍스트 wrap | 버튼 `white-space:nowrap;flex-shrink:0`, 필터 `flex:1;min-width:0`(칩은 overflow 클립) |
| 3 | **글쓰기 버튼 중복** (PC 정보: 필터+레일) | 필터행·레일 양쪽 배치 | PC 메인은 칩만, 글쓰기는 레일에만 |
| 4 | **"컴퓨터공학 · 3/학년" 줄바꿈** (PC 팀원 학생카드) | 2열이라 카드가 좁음 | 학생 카드는 내용 많아 **PC 메인 1열(넓게)**. 게시글(짧음)은 2열 유지 |

**검증(크롬 실측)**: `.pbody` display:flex 복구·사이드바 412px 전체높이·사이드바\|본문 같은 행(라이트·다크 both), 필터 글쓰기 28px 단일행, 학생 헤더 15px 단일행. 라이트/다크 × 정보/팀원 × 모바일/PC × 데이터/빈 전 조합 정상.

**교훈**: 목업 스캐폴드 재사용 시 **클래스 이름 충돌 금지**(프레임 `.pbody` vs 텍스트 `.pbody`). 실제 앱은 Tailwind라 이 이슈는 목업 한정. 우측 버튼+가변 텍스트 카드는 PC 2열에서 폭 부족 → 1열 권장.
