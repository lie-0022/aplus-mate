# 시간표 플래너 + "봐주세요" 게시판 — 설계

> 2026-07-11 확정. 범위(사용자 결정): **짜보는 플래너** + **봐주세요 게시판** + **에타 시간표 이미지 가져오기**.
> 매칭 재설계 이후 가장 큰 단일 기능 — 여러 세션 분량.

## 배경·결정
- 에타에 "시간표 봐주세요" 글 수요가 큼. 이걸 A+ Mate 안으로.
- **성격 = 짜보는 플래너**(수강신청 전 다음 학기를 미리 구성하는 가상 시간표). 실제 수강 등록(userCourses)과 **별개**. 한 명이 여러 개·여러 학기.
- 기존 "내 시간표"(/timetable, 수강 등록 기반)는 유지. 플래너에서 "이대로 실제 등록"으로 연결.
- **에타 데이터 크롤링은 안 함**(법적·전략적 리스크: 에타는 크롤러에 법적 대응 이력, 강의평은 DB권리·정보통신망법 보호 대상, 경쟁사 데이터로 자기 DB 채우는 건 전형적 분쟁형태, 우리 가치=오리지널 후기 오염). **유일하게 정당한 각도 = 유저가 "자기" 에타 시간표 이미지를 직접 업로드**(본인 데이터·에타 서버에 요청 안 함).

## 데이터 모델 (신규)
```
timetables        { id, userId, semester(varchar), title, postedAt(nullable=비공개 초안), createdAt, updatedAt }
timetableItems    { id, timetableId, courseId(nullable=카탈로그 연결), title, dayOfWeek, startPeriod, endPeriod, room(nullable), cyber(bool), colorIndex(nullable) }
timetableComments { id, timetableId, userId, content, createdAt }   // 봐주세요 댓글, 익명 노출
```
- **item = 연속 교시 한 블록**(목4,5 = 한 행 / 화2,3 = 또 한 행). courseSchedules(교시당 1행)를 담을 때 연속 구간으로 접어 저장 → 표시 쉽고 스냅샷처럼 고정됨(카탈로그 바뀌어도 저장한 플랜 불변).
- courseId 있으면 "실제 수강 등록" 연결 가능. 커스텀 블록·에타 반입분은 courseId=null.

## 격자 컴포넌트 추출 (선수 리팩터)
- `client/src/pages/Timetable.tsx`의 격자 렌더(요일×교시, 연강 접기, lane 분할, 색)를 **`<TimetableGrid>`**로 추출.
- 3곳에서 재사용: /timetable(수강, 읽기), 플래너(편집), 게시판 보기(읽기). 모드 = read-only | editable.

## 로드맵 (R7 — R6=2026-2 적재와 맞물림)
- **A. 모델 + 격자 추출** — 마이그레이션 3종, `<TimetableGrid>` 추출·기존 화면 무회귀 확인.
- **B. 플래너** — 학기 선택 → 수업 검색·담기(courseSchedules→items 확장) + 커스텀 블록 + **시간충돌 경고**(이미 데이터 있어 쉬움) + 저장(여러 개). "이대로 실제 수강 등록" 버튼(courseId 있는 항목 → userCourses, 학기 일치 시).
- **C. 봐주세요 게시판** — 공개 토글(postedAt) → `/timetables` 목록(학기 필터·익명) → 격자 보기 + 댓글. 내비 추가.
- **D. 에타 이미지 가져오기(최종·리스크 높음)** — 본인 에타 시간표 **이미지 업로드** → 격자 옆에 참고로 띄우고 **수동 배치 보조**가 1차. 자동 파싱(격자 OCR)은 fragile(색블록·한글·좌표) → 여력 될 때 별도 검토. 크롤링 아님을 UI에도 명시.

## 의존성·주의
- **다음 학기(2026-2) 카탈로그가 있어야** 플래너에서 그 학기 수업을 담을 수 있음 → R6 적재와 맞물림. 그 전엔 2026-1로 시연·테스트.
- 봐주세요 글도 익명(수락 전 매칭·리뷰와 동일 정책). 소규모 특정 방지 안내.
- 이미지 업로드 = 파일 저장 필요(현재 스택엔 파일 스토리지 없음 → 용량 작으니 DB base64 or 외부 무료 스토리지 검토).
