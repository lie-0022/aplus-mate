# A+ Mate 디자인 시스템 (2026-07 리브랜딩)

> 방향: **C 볼드 캠퍼스** 파생 → 모바일 시안 D(미드나잇 네온)+E(소프트 클레이)를
> **하나의 앱**으로 통합. **라이트=소프트 클레이(기본) / 다크=딥 인디고(야간)**.
> 로고: `A+ Mate` 이탤릭 스피드라인(보라→블루 그라데이션). 라이트/다크 변형 PNG.

## 0. 원칙 (바닥선)
- **가독성 최우선**: 본문 15px · 보조 13px · **라벨/뱃지 최소 12px**(9~11px 금지).
- **모노스페이스 폰트 금지**(사용자 명시). 폰트는 **Pretendard(본문) + Archivo(로고 전용)** 2종.
- **태그·뱃지에 브랜드 보라 남발 금지** → 중립색(`--tag-*`). 보라는 CTA·강조·로고에만.
- **버튼은 테두리 없이 채움**(솔리드/은은한 채움 칩).
- **밀도 2모드**: 표현적(대시보드·랜딩) / 차분(교수·운영자·팀상세·설문 — 파스텔·글로우 절제, 글자 크기는 유지).

## 5. 레이아웃 기준 — 모바일과 PC는 다르다 (2026-07-06 확정)

**모바일 (기조 컨펌됨):**
- 상단 헤더바(`bg-card`, 로고·알림·아바타) + 콘텐츠 + 하단 탭 5개(`bg-card`).
- 콘텐츠 = 카드 세로 스택. 소프트 섀도우(라이트)/미세 보더(다크). 라운드 16~20.
- 핵심 액션 카드(예: 수업 코드 참여)는 `--secondary` 채움 + 아이콘 배지로 강조.

**PC (모바일틱 금지 — 데스크톱 고유 패턴):**
- ❌ 모바일 컴포넌트를 가운데 좁은 컬럼으로 정렬만 하는 것 금지.
- ✅ 기본 패턴 = **메인 + 우측 레일** (대시보드에서 확립, 전 화면 일관 적용):
  - 메인(≈1.6fr): 화면의 주 목록/콘텐츠. **목록은 2열 카드 그리드**로 넓이 활용.
  - 우측 레일(≈1fr): 빠른 액션·입력 카드·안내 (예: 수업 코드 참여, 새 수업 만들기, 추천).
- 상단 헤더(전체폭) + 좌측 사이드바(AppLayout 기존 구조) 위에 위 패턴을 얹는다.
- 남는 공간은 그리드/레일/여백으로 소화 — **글자 축소 금지**.
- 정보 많은 화면(교수·운영자)은 다열·테이블 허용(차분 밀도), 글자 크기는 유지.

## 6. 목업·컨펌 워크플로우 (화면 단위 리디자인)
- 화면 하나씩: **기존 vs 수정본** 목업 → 사용자 컨펌 → 실제 코드 반영 → 배포.
- 목업 매트릭스: 라이트/다크 × 데이터 있음/빈 상태 × 모바일/PC.
- **목업은 실제 보이는 화면 전체**(모바일: 헤더바+탭바 포함 / PC: 헤더+사이드바 포함).
- 프레임 비율: **모바일 9:19.5**(340×737), **PC 16:10**(720×450). 임의 비율 금지.
- 목업 검증·컨펌은 사용자가 직접(브라우저/프리뷰). 로고는 data URI 임베드.
